# 黑马点评 28 天进阶计划（基础薄弱版，含验证点/缓冲/类比/面试问法）
每天 1.5–3 小时，四周完成。所有学习点写清楚“看什么、干什么、面试问什么”，并给常见坑速查表。

类比小抄（贯穿使用，口头背诵）
- 分层：Controller / Service / Mapper = 餐厅前台 / 后厨 / 采购。前台接单，不做菜；后厨做菜；采购对接仓库（DB/Redis）。
- 分布式锁：公共厕所钥匙，谁先拿钥匙谁进，钥匙有超时时间，防止拿走不还。
- Token：超市购物券，入口领券（登录），之后凭券购物（访问接口），过期需重新领。
- 缓存击穿：热门餐厅一次只让一个厨师补菜，其它人排队；逻辑过期=先上旧菜并异步备新菜；互斥锁=只允许一个人去补菜。
- MQ：取号排队，先领号（生产），等叫号（消费），避免所有人挤前台。
- GEO：地图导航，按距离排序的附近餐厅列表。

第 1 周（Day1–7）：环境与基础夯实 + 缓冲
- Day1 环境基线  
  看：pom.xml、.mvn/jvm.config（JDK21 运行，target/release=11）；docker-compose.yml。  
  干：`mvn -Dmaven.test.skip=true clean package`；`docker-compose up -d`；导数时若报 `Invalid default value`，用 `--init-command="SET SESSION sql_mode='STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION';"` 导入 `src/main/resources/db/hmdp.sql`。  
  验证：后端 8081 可访问 `/shop-type/list`。  
  面试：问你环境如何落地、sql_mode 兼容问题怎么解决。
- Day2 前端基线  
  看：frontend/hmdp/package.json。  
  干：Node>=18（nvm-windows 装 20.x），`cd frontend/hmdp && rm -rf node_modules package-lock.json && npm i && npm run dev -- --host --port 5173`。  
  验证：首页可开，无 CORS；CORS 配置在 `src/main/java/com/hmdp/config/MvcConfig.java`。  
  面试：跨域怎么解决？前后端端口多少？
- Day3 分层与日志  
  看：`src/main/java/com/hmdp/controller/UserController.java`、`ShopController.java`；`service/impl/UserServiceImpl.java`、`ShopServiceImpl.java`；`mapper` 层；`application.yaml` 日志配置。  
  干：画“请求→前台/后厨/采购”链路；改一条日志确认 `logs/app.log` 有输出。  
  面试：分层职责？Service 该不该写 SQL？
- Day4 Redis 基础  
  看：`utils/RedisConstants.java`、`utils/CacheClient.java`、`utils/RedisData.java`。  
  干：redis-cli 读写验证码 key、shop 缓存 key，理解 TTL/结构。  
  面试：key 命名规范？CacheClient 封装意义？
- Day5 拦截器与认证骨架  
  看：`interceptor/RefreshTokenInterceptor.java`（刷新 token）、`LoginInterceptor.java`（鉴权）、`utils/UserHolder.java`（ThreadLocal）。  
  干：加日志看拦截顺序；未带 token 访问受限接口，确认被拦截。  
  面试：为什么两层拦截？ThreadLocal 为什么要 remove？
- Day6 缓冲日/复盘  
  干：整理疑问，补查资料。
- Day7 校准  
  干：口述 3 分钟登录流程，画时序图。

常见坑速查表
- SQL 导入：报 `Invalid default value` → 用 `--init-command="SET SESSION sql_mode='STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION';"` 导入，或改 `begin_time/end_time` 默认值为 `CURRENT_TIMESTAMP`；库名需与 `application.yaml` 一致。
- CORS：前端 5173/5174 访问后端 8081 报跨域 → `src/main/java/com/hmdp/config/MvcConfig.java` 已放行，重启后端。
- Node 版本：Vite5 需 Node>=18，低版本报 rollup/crypto 错误。
- JDK/编译：target/release=11 避免 “Unsupported class file major version 65”；运行 JDK21。全 21 时同步 pom 与 `.mvn/jvm.config`。
- 端口：后端 8081、前端 5173/5174，冲突换端口或关占用进程。
- 登录/验证码：看控制台 + `logs/app.log` → 查 Redis 验证码 key、token hash → 查 DB 用户表。
- 秒杀：检查 Redis 秒杀券数据、Lua 校验、一人一单、MQ 是否运行。
- Feed/点赞：检查 Redis ZSet/Set 是否写入；确认已登录、token 有效。
