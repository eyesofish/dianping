## 1. 短信登录链路（最基础必须会）

### 1.1 发送验证码 `/user/code`
**你要做的**：输入手机号，系统生成 6 位验证码，存进 Redis。  
**源码位置**：  
- Controller：`src/main/java/com/hmdp/controller/UserController.java` → `sendCode()`  
- Service：`src/main/java/com/hmdp/service/impl/UserServiceImpl.java` → `sendCode()`  
- 校验工具：`src/main/java/com/hmdp/utils/RegexUtils.java`  
- Redis key 常量：`src/main/java/com/hmdp/utils/RedisConstants.java`  

**傻瓜步骤**：  
1) 调接口：`POST http://localhost:8081/user/code?phone=13800138000`  
2) 看 Redis：会出现 `login:code:13800138000`，TTL 2 分钟  
3) 日志里会打印验证码（`UserServiceImpl.sendCode` 的 `log.info`）  

**你要记住**：验证码只是临时存 Redis，不存数据库。

### 1.2 登录 `/user/login`
**你要做的**：用手机号 + 验证码换 token。  
**源码位置**：  
- Controller：`src/main/java/com/hmdp/controller/UserController.java` → `login()`  
- Service：`src/main/java/com/hmdp/service/impl/UserServiceImpl.java` → `login()`  
- DTO：`src/main/java/com/hmdp/dto/LoginFormDTO.java`、`src/main/java/com/hmdp/dto/UserDTO.java`  

**傻瓜步骤**：  
1) 调接口：  
   ```json
   POST http://localhost:8081/user/login
   {"phone":"13800138000","code":"123456"}
   ```  
2) 后端做的事：  
   - 从 Redis 取验证码对比  
   - 查库是否有用户，没有就创建  
   - 生成 token（`UUID.randomUUID()`）  
   - 把用户信息放进 Redis Hash（key：`login:token:<token>`）  
3) 返回 token 给前端  

**重点**：用户信息存 Redis Hash，不是 Session。  
**路径**：`UserServiceImpl.login` 里 `stringRedisTemplate.opsForHash().putAll(...)`

### 1.3 拦截器校验登录
**你要做的**：后续请求带 `authorization` 头部，系统才能认为你已登录。  
**源码位置**：  
- 注册拦截器：`src/main/java/com/hmdp/config/MvcConfig.java` → `addInterceptors()`  
- 刷新 token：`src/main/java/com/hmdp/interceptor/RefreshTokenInterceptor.java`  
- 鉴权拦截：`src/main/java/com/hmdp/interceptor/LoginInterceptor.java`  
- ThreadLocal：`src/main/java/com/hmdp/utils/UserHolder.java`

**傻瓜步骤**：  
1) 不带 token 访问 `/user/me` 会 401  
2) 带 `authorization: <token>` 再访问就成功

## 2. 店铺查询缓存（减轻数据库压力）

### 2.1 查询店铺 `/shop/{id}`
**你要做的**：访问店铺详情，优先走 Redis 缓存。  
**源码位置**：  
- Controller：`src/main/java/com/hmdp/controller/ShopController.java` → `queryShopById()`  
- Service：`src/main/java/com/hmdp/service/impl/ShopServiceImpl.java` → `queryById()`  
- 缓存模板：`src/main/java/com/hmdp/utils/CacheClient.java`  
- 逻辑过期包装：`src/main/java/com/hmdp/utils/RedisData.java`  RedisData 有两个东西，过期事件和数据组合，@Data 会提供setData和getData方法
开始的时候需要预热redis，如果没有命中缓存，就查mysql然后重建缓存

下一步修改计划，把防止穿透和解决缓存击穿（热点过期瞬间大量并发打 DB）都用起来，现在只用了解决缓存击穿
**傻瓜步骤**：  
1) 调接口：`GET http://localhost:8081/shop/1`  
2) Redis 会出现 `cache:shop:1`  
3) 再次访问会更快（缓存命中）  

**关键点**：  
`ShopServiceImpl.queryById` 调用 `CacheClient.queryWithLogicalExpire`，命中直接返回；过期会异步重建缓存。

### 2.2 缓存问题怎么防
**缓存穿透（查不到的 id）**  
- 处理逻辑：`CacheClient.queryWithPassThrough`  
- 做法：查库为空时，缓存空字符串，TTL 短一点。  

**缓存击穿（热门 key 瞬间过期）**  
- 处理逻辑：`CacheClient.queryWithLogicalExpire`  
- 做法：逻辑过期 + 线程池异步重建。  

**缓存雪崩（大批 key 同时过期）**  
- 目前代码里是固定 TTL（`RedisConstants.CACHE_SHOP_TTL`）。  
- 如果要随机 TTL，需要在 `CacheClient.set` 或 `ShopServiceImpl` 自己加逻辑（源码里暂未实现）。

## 3. 秒杀券（高并发核心）

### 3.1 新建秒杀券 `/voucher/seckill`
**你要做的**：创建秒杀券并把库存写入 Redis。  
**源码位置**：  
- Controller：`src/main/java/com/hmdp/controller/VoucherController.java` → `addSeckillVoucher()`  
- Service：`src/main/java/com/hmdp/service/impl/VoucherServiceImpl.java` → `addSeckillVoucher()`  
- Redis key：`RedisConstants.SECKILL_STOCK_KEY`

**关键点**：  
创建后会执行：`set seckill:stock:<voucherId> <stock>`。

### 3.2 抢购 `/voucher-order/seckill/{id}`
**你要做的**：抢购秒杀券，先 Lua 预检，再异步下单。  
**源码位置**：  
- Controller：`src/main/java/com/hmdp/controller/VoucherOrderController.java` → `seckillVoucher()`  
- Service：`src/main/java/com/hmdp/service/impl/VoucherOrderServiceImpl.java` → `seckillVoucher()`  
- Lua 脚本：`src/main/resources/seckill.lua`  
- ID 生成：`src/main/java/com/hmdp/utils/RedisIdWorker.java`

**Lua 做了什么**（见 `seckill.lua`）：  
- 检查库存 `seckill:stock:<voucherId>`  
- 判断是否重复下单（Set：`seckill:order:<voucherId>`）  
- 扣减库存、记录下单、写入 stream（历史方案）  

### 3.3 RabbitMQ 异步下单
**你要做的**：下单逻辑丢给 MQ，系统异步处理。  
**源码位置**：  
- 队列/交换机：`src/main/java/com/hmdp/config/QueueConfig.java`  
- 发送消息：`VoucherOrderServiceImpl.seckillVoucher()`  
- 消费消息：`src/main/java/com/hmdp/listener/SeckillVoucherListener.java`  
- MQ 连接配置：`src/main/resources/application.yaml`

**流程**：  
1) 预检通过后，`VoucherOrderServiceImpl` 把订单消息发到交换机 X（routingKey=XA）。  
2) `SeckillVoucherListener` 监听队列 QA/QD，保存订单并扣减库存。  

### 3.4 分布式锁（防重复下单）
**源码位置**：  
- Redisson 配置：`src/main/java/com/hmdp/config/RedissonConfig.java`  
- 业务加锁：`VoucherOrderServiceImpl.handleVoucherOrder()`  
- 自研锁：`src/main/java/com/hmdp/utils/SimpleRedisLock.java`（备用方案）  

**你要记住**：当前 MQ 方案主要走 `SeckillVoucherListener` 下单逻辑；若要严格“一人一单”，可参考 `handleVoucherOrder` 和 `createVoucherOrder` 的事务实现。

## 5. 附近店铺（Redis GEO）

### 查询 `/shop/of/type`
**源码位置**：  
- Controller：`src/main/java/com/hmdp/controller/ShopController.java` → `queryShopByType()`  
- Service：`src/main/java/com/hmdp/service/impl/ShopServiceImpl.java` → `queryShopByType()`

**关键点**：

- 手动 `GEOADD shop:geo:<typeId> <lng> <lat> <shopId>`  
或  
- 在 `ShopServiceImpl` / 初始化脚本里补充写入逻辑。

## 6. Redis 数据结构在项目里的落点（全是源码对照）

- **String**：验证码、秒杀库存  
  - `UserServiceImpl.sendCode`  
  - `VoucherServiceImpl.addSeckillVoucher`  
- **Hash**：登录用户信息  
  - `UserServiceImpl.login`  
- **Set**：关注关系、秒杀用户下单  
  - `FollowServiceImpl.follow`  
  - `seckill.lua`（`seckill:order:<voucherId>`）  
- **ZSet**：点赞、Feed 流  
  - `BlogServiceImpl.updateLike`  
  - `BlogServiceImpl.saveBlog` / `quertBlogOfFollow`  
- **GEO**：附近店铺  
  - `ShopServiceImpl.queryShopByType`  
- **Bitmap**：签到  
  - `UserServiceImpl.sign` / `signCount`  
- **Stream（旧方案）**：秒杀订单流  
  - `seckill.lua` 写入 `stream.orders`  
  - `VoucherOrderServiceImpl` 里 Stream 消费逻辑已注释（历史方案）

## 7. 常见卡点排错（对着源码找）
- **401 未登录**：检查 `authorization` 头，拦截逻辑在 `RefreshTokenInterceptor` / `LoginInterceptor`。  
- **秒杀库存始终为空**：先调用 `/voucher/seckill` 写入 Redis 库存（`VoucherServiceImpl.addSeckillVoucher`）。  
- **MQ 不工作**：确认 `docker-compose.yml` 起了 RabbitMQ，队列配置在 `QueueConfig`，连接在 `application.yaml`。  
- **附近店铺无结果**：先写 GEO 数据（源码只查不写）。
